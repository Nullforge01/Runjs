Build a Real Todo App

You've learned variables, functions, arrays, objects, DOM, events, async, and localStorage. Time to combine everything.

Project Requirements

Build a todo app that does this:

  1. Add todos: Type in input, click Add, todo appears in list
  2. Mark done: Click todo to toggle complete. Add class "done"
  3. Delete todos: Each todo has a delete button
  4. Persist data: Todos save to localStorage. Reload keeps them
  5. Filter: Show All, Active, or Completed todos
  6. Counter: Show "X items left" for active todos

Data Structure

Store todos as array of objects:

const todos = [
  { id: 1, text: "Learn JS", done: false },
  { id: 2, text: "Build app", done: true }
];

// 1. State
let todos = JSON.parse(localStorage.getItem("todos") || "[]");
let filter = "all"; // all, active, completed

// 2. Save state
function saveTodos() {
  localStorage.setItem("todos", JSON.stringify(todos));
}

// 3. Render UI
function render() {
  const list = document.getElementById("todo-list");
  const filtered = todos.filter(t => {
    if (filter === "active") return !t.done;
    if (filter === "completed") return t.done;
    return true;
  });
  
  list.innerHTML = filtered.map(t => `
    <li data-id="${t.id}" class="${t.done ? 'done' : ''}">
      <span>${t.text}</span>
      <button class="delete">×</button>
    </li>
  `).join("");

}

Event Delegation Tip

Don't add listeners to each todo. Listen on parent list once:

document.getElementById("todo-list").addEventListener("click", (e) => {
  const li = e.target.closest("li");
  if (!li) return;
  const id = Number(li.dataset.id);
  
  if (e.target.classList.contains("delete")) {
    deleteTodo(id);
  } else {
    toggleTodo(id);
  }
});

Common Gotchas

  • ID generation: Use Date.now() or increment counter for unique IDs
  • Re-render after change: Always call render() after modifying todos
  • XSS: Use textContent for user input, not innerHTML
  • Empty input: Check input.value.trim() === "" before adding
