1. Events: User Actions

Events fire when users interact: click, type, submit, hover. You listen for them and run code in response.

const btn = document.getElementById("btn");

btn.addEventListener("click", () => {
  console.log("Clicked!");
});

2. Common Event Types

EventFires WhenCommon Element
clickMouse clickbutton, div
inputUser types/changes valueinput, textarea
submitForm submittedform
changeValue changed + blurselect, checkbox
keydownKey presseddocument, input

3. The Event Object

Every listener gets an event object with details.

document.addEventListener("click", (e) => {
  console.log(e.target); // What was clicked
  console.log(e.clientX, e.clientY); // Mouse position
});

input.addEventListener("keydown", (e) => {
  console.log(e.key); // "Enter", "a", "Escape"
  if (e.key === "Enter") {
    console.log("Enter pressed");
  }
});

4. Forms & preventDefault

Forms reload the page by default on submit. Stop it with preventDefault().

<form id="login">
  <input id="user" type="text">
  <button type="submit">Login</button>
</form>

<script>
const form = document.getElementById("login");
const input = document.getElementById("user");

form.addEventListener("submit", (e) => {
  e.preventDefault(); // Stop page reload
  const username = input.value.trim();
  if (username === "") {
    alert("Enter username");
    return;
  }
  console.log("Logging in:", username);
});
</script>

5. Form Validation Patterns

const emailInput = document.getElementById("email");
const errorBox = document.getElementById("error");

emailInput.addEventListener("input", () => {
  const email = emailInput.value;
  if (!email.includes("@")) {
    errorBox.textContent = "Invalid email";
  } else {
    errorBox.textContent = "";
  }
});

6. Event Delegation

Listen on parent instead of 100 child buttons. Works for dynamic elements.

<ul id="list">
  <li><button>Delete 1</button></li>
  <li><button>Delete 2</button></li>
</ul>

<script>
const list = document.getElementById("list");

list.addEventListener("click", (e) => {
  if (e.target.tagName === "BUTTON") {
    e.target.parentElement.remove(); // Delete the li
  }
});
</script>

7. Common Mistakes

  • Forgetting preventDefault: Form submits and page reloads, JS state lost
  • Multiple listeners: Adding listener in a loop without removing old ones stacks them
  • this in arrow functions: Arrow functions don’t bind this. Use e.target instead
  • Accessing.value too early: Get input.value inside the event, not outside

8. Real Example: Live Search

<input id="search" placeholder="Type to filter">
<ul id="items">
  <li>Apple</li>
  <li>Banana</li>
  <li>Cherry</li>
</ul>

<script>
const search = document.getElementById("search");
const items = document.querySelectorAll("#items li");

search.addEventListener("input", () => {
  const term = search.value.toLowerCase();
  items.forEach(li => {
    const match = li.textContent.toLowerCase().includes(term);
    li.style.display = match? "" : "none";
  });
});
</script>

Mastery Checklist:

  • Attach events with addEventListener
  • Use the event object and e.target
  • Stop form reload with preventDefault()
  • Read form values with .value
  • Validate input on input or submit
  • Use event delegation for dynamic elements
