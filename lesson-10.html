Lesson 10: Async JS & Promises

medium · 60 XP

1. Sync vs Async

JS runs one thing at a time. Sync code blocks. Async code says "start this, I'll check back later".

// Sync - blocks
console.log("1");
alert("Blocks here"); // Page freezes
console.log("3");

// Async - non-blocking
console.log("1");
setTimeout(() => console.log("2"), 1000);
console.log("3");
// Output: 1, 3, 2

2. Callbacks: The Old Way

Pass a function to run later. Leads to "callback hell" when nested.

function getUser(id, callback) {
  setTimeout(() => {
    callback({ id, name: "Alex" });
  }, 1000);
}

getUser(1, (user) => {
  console.log(user.name); // "Alex" after 1s
});

3. Promises: Better Async

Promise = placeholder for future value. 3 states: pending, fulfilled, rejected.

const fetchData = new Promise((resolve, reject) => {
  setTimeout(() => {
    const success = true;
    if (success) {
      resolve("Data loaded");
    } else {
      reject("Error loading");
    }
  }, 1000);
});

fetchData
  .then(data => console.log(data)) // "Data loaded"
  .catch(err => console.log(err));

4. Chaining Promises

function getUser(id) {
  return Promise.resolve({ id, name: "Alex" });
}

function getPosts(userId) {
  return Promise.resolve(["Post 1", "Post 2"]);
}

getUser(1)
  .then(user => getPosts(user.id))
  .then(posts => console.log(posts)) // ["Post 1", "Post 2"]
  .catch(err => console.log("Failed:", err));

5. async/await: Modern Syntax

Write async code like sync. Must be inside async function. Use try/catch for errors.

async function loadUser() {
  try {
    const user = await getUser(1); // waits here
    const posts = await getPosts(user.id);
    console.log(user.name, posts);
  } catch (err) {
    console.log("Error:", err);
  }
}

loadUser();

6. fetch() — Real API Calls

fetch returns a Promise. Must await twice: once for response, once for JSON.

async function getJoke() {
  const res = await fetch("https://api.chucknorris.io/jokes/random");
  const data = await res.json();
  console.log(data.value);
}

getJoke();

7. Promise.all — Run in Parallel

async function loadAll() {
  const [user, posts] = await Promise.all([
    getUser(1),
    getPosts(1)
  ]);
  console.log(user, posts);
}
// Faster than awaiting one by one

8. Common Mistakes

  • Forgetting await: const data = fetch(url) gives Promise, not data. Use await fetch(url)
  • No try/catch: Unhandled rejection crashes in async/await
  • Blocking with await: await x; await y; runs serial. Use Promise.all for parallel
  • Not returning in .then: Breaks chain. Always return for next .then

9. Real Example: Load Profile

async function loadProfile(userId) {
  try {
    const res = await fetch(`/api/users/${userId}`);
    if (!res.ok) throw new Error("User not found");
    const user = await res.json();
    document.getElementById("name").textContent = user.name;
  } catch (err) {
    document.getElementById("error").textContent = err.message;
  }
}
