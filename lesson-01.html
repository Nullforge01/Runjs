Lesson 1: Variables & Data Types

easy · 25 XP

1. What Are Variables?

Variables are containers that store data values. Think of them as labeled boxes where you keep stuff for later.

let userName = "Alex";
const userAge = 25;

2. Declaring Variables: let vs const vs var

KeywordReassign?ScopeUse When
constNoBlockValue won't change. Default choice.
letYesBlockValue will change later.
varYesFunctionAvoid. Old, causes bugs.

Rule: Use const by default. Use let only if you plan to reassign. Never use var.

3. Primitive Data Types

String — Text

Any text wrapped in quotes. Single ' ', double " ", or backticks ` `.

const firstName = "Sarah";
const lastName = 'Connor';
const greeting = `Hello, ${firstName}`; // Template literal

Number — Math Values

Integers and decimals. No quotes.

const age = 30;
const price = 19.99;
const negative = -5;

Boolean — True/False

Used for logic, conditions, flags.

const isLoggedIn = true;
const hasPermission = false;

Undefined & Null — Empty

let x; // x is undefined - declared but no value
const y = null; // y is null - intentionally empty

4. Common Mistakes

  • Forgetting quotes: const name = Alex → Error. Should be "Alex"
  • Reassigning const: const x = 1; x = 2; → Error
  • Numbers in quotes: "5" is a string, not number. "5" + 1 = "51"

5. Real Examples

// User profile
const username = "dev_pro";
let postCount = 0;
const isVerified = true;

// Product
const productName = "Laptop";
const productPrice = 999;
let inStock = true;

// Update values
postCount = postCount + 1; // let allows this
// username = "hacker"; // Error: const can't change

  • Know when to use const vs let
  • Identify String, Number, Boolean types
  • Avoid var and common errors
