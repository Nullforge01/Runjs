Lesson 9: Array Methods

medium · 50 XP

1. Why Array Methods?

For loops work, but array methods are shorter, clearer, and less error-prone. They express intent: map = transform, filter = pick, reduce = combine.

// For loop way
const nums = [1, 2, 3];
const doubled = [];
for (let i = 0; i < nums.length; i++) {
  doubled.push(nums[i] * 2);
}

// Array method way
const doubled = nums.map(n => n * 2); // [2, 4, 6]

2. map — Transform Every Item

Returns a new array. Same length. Changes each item.

const prices = [10, 20, 30];
const withTax = prices.map(p => p * 1.1);
// [11, 22, 33]

const users = [{name: "Alex"}, {name: "Sam"}];
const names = users.map(u => u.name);
// ["Alex", "Sam"]

3. filter — Keep Some Items

Returns new array. Can be shorter. Keeps items where callback returns true.

const scores = [55, 90, 60, 85];
const passed = scores.filter(s => s >= 70);
// [90, 85]

const todos = [
  {text: "Code", done: true},
  {text: "Sleep", done: false}
];
const done = todos.filter(t => t.done);
// [{text: "Code", done: true}]

4. reduce — Combine Into One Value

Most powerful. Takes accumulator + current item. Returns single value.

const nums = [1, 2, 3, 4];
const sum = nums.reduce((total, n) => total + n, 0);
// 0 + 1 + 2 + 3 + 4 = 10

const cart = [
  {price: 10, qty: 2},
  {price: 5, qty: 3}
];
const total = cart.reduce((sum, item) => sum + item.price * item.qty, 0);
// 10*2 + 5*3 = 35

Params: reduce((accumulator, current) => {}, initialValue)

5. find & findIndex — Get One Item

const users = [
  {id: 1, name: "Alex"},
  {id: 2, name: "Sam"}
];

const user = users.find(u => u.id === 2);
// {id: 2, name: "Sam"}

const idx = users.findIndex(u => u.name === "Alex");
// 0
// Returns undefined or -1 if not found

6. forEach — Do Something With Each

Like a for loop. No return value. Use for side effects.

const logs = ["error", "warn", "info"];
logs.forEach((log, i) => {
  console.log(`${i}: ${log}`);
});
// 0: error, 1: warn, 2: info

7. some & every — Test Items

const nums = [2, 4, 6, 7];
nums.some(n => n % 2 === 1); // true - at least one odd
nums.every(n => n > 0); // true - all positive

8. Chaining Methods

Combine them. Each returns an array, so you can call another.

const data = [1, 2, 3, 4, 5, 6];

const result = data
 .filter(n => n % 2 === 0) // [2, 4, 6]
 .map(n => n * 10) // [20, 40, 60]
 .reduce((sum, n) => sum + n, 0); // 120

9. Common Mistakes

  • Forgetting return: arr.map(x => { x * 2 }) returns [undefined, undefined]. Use x => x * 2 or { return x * 2 }
  • Mutating in map: Don't arr.map(x => arr.push(x)). Map should be pure
  • reduce without initial: [].reduce((a,b) => a+b) crashes. Always pass initial value
