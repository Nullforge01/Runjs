Lesson 4: Loops

easy · 35 XP

1. Why Loops?

Loops repeat code without copy-pasting. Need to print 1 to 100? Loop it. Need to check every item in a list? Loop it.

// Without loop - bad
console.log(1);
console.log(2);
console.log(3);
// ... 97 more lines

// With loop - good
for (let i = 1; i <= 100; i++) {
  console.log(i);
}

2. for Loop — When You Know How Many Times

for (let i = 0; i < 5; i++) {
  console.log(i);
}
// Output: 0, 1, 2, 3, 4

3 parts:

  1. let i = 0 → Start. Runs once
  2. i < 5 → Condition. Check before each loop
  3. i++ → Update. Runs after each loop

3. while Loop — When Condition Matters

Runs as long as condition is true. You control the update inside.

let count = 0;

while (count < 3) {
  console.log(count);
  count++; // Don't forget or infinite loop!
}
// Output: 0, 1, 2

4. for...of — Loop Through Arrays

Easiest way to touch every item in an array.

const fruits = ["apple", "banana", "cherry"];

for (const fruit of fruits) {
  console.log(fruit);
}
// apple, banana, cherry

5. break & continue

// break — exit loop early
for (let i = 0; i < 10; i++) {
  if (i === 5) break; // stops at 5
  console.log(i);
}
// Output: 0,1,2,3,4

// continue — skip this round
for (let i = 0; i < 5; i++) {
  if (i === 2) continue; // skip 2
  console.log(i);
}
// Output: 0,1,3,4

6. Common Mistakes

  • Infinite loop: while(true) with no break. Crashes browser
  • Off-by-one: i <= 5 vs i < 5. One runs 6 times, other runs 5
  • Modifying array while looping: Can skip items. Loop backwards or use filter instead

7. Real Examples

// Sum all numbers 1 to 10
let sum = 0;
for (let i = 1; i <= 10; i++) {
  sum += i;
}
console.log(sum); // 55

// Find first even number
const nums = [1, 3, 4, 7, 8];
for (const n of nums) {
  if (n % 2 === 0) {
    console.log("First even:", n); // 4
    break;
  }
}

// Countdown
let n = 3;
while (n > 0) {
  console.log(n);
  n--;
}
console.log("Go!");

Mastery Checklist:

  • Write for loops with start, condition, update
  • Use while for condition-based loops
  • Loop arrays with for...of
  • Control flow with break and continue
  • Avoid infinite loops and off-by-one errors
